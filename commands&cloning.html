GitHUb Commands & Cloning Process

Commands in GitHub

  • Git init -- It creates an empty git Repository in local Initialization.
  • Git Status -- It checks whether our files tracking/untracking.
  • Git Add ./filename/foldername -- To change the file to Tracking.
  • Git Commit -m "message related to file" -- Version message.
  • Git Remote Add Origin "http url" -- To link the GitHub repository and Git.
  • Git Branch -M Main -- To shift the branch from master to main.
  • Git Push Origin Main -- To upload our local files in GitHub.
  • Git Config -- global user.name "user_name"
  • Git Config -- global user.email "user_email"

W have to give These 2 Configurations when we are adding our first files from local to remote.

Clonig Process

When we are working on projects different persons make updates in the file so we want to get that updated files we have to do cloning process for getting files from remote to local

  • Git Clone "http url" -- For Cloning file
  • Git Add. -- For accessing updated file
  • Git Commit -m " updated version"
  • Git Pull Origin Main -- For getting file from remote to local.
